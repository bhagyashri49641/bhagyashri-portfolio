
# Customization guide

## Replace placeholder content first
Update:
- name
- role
- tagline
- about text
- email
- GitHub and LinkedIn links
- projects
- experience
- resume file in `frontend/public/resume.pdf`

## Good future upgrades
- Add Framer Motion animations
- Add blog section using MDX
- Add admin CMS later
- Add PostgreSQL and contact form submissions
- Add project detail pages with dynamic routes
- Add dark/light toggle later if you want multiple themes

## UI direction suggestions
For your taste, keep:
- dark theme as default
- pink/lavender/violet pastel highlights
- soft borders and blur, but not too much
- large section spacing
- rounded cards
- project screenshots in future
- elegant typography

## Content tips
Use a project card format like:
- Problem
- What you built
- Tech stack
- Result / impact

## Best next steps
1. Get this starter running locally.
2. Replace all placeholder content.
3. Add 2 to 4 strong projects.
4. Add resume PDF.
5. Push backend and frontend separately to GitHub if needed.
6. Deploy backend on Render.
7. Deploy frontend on Vercel.
8. Update production environment variables.
